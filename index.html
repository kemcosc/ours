# Ours Peptides — Website A fast, static storefront (no server required) with its own cart and checkout. Deploys by drag-and-drop to Netlify Drop, just like before. --- ## ⚠️ Before you go live — 3 things to do ### 1. Set your real prices All prices are currently placeholders showing **$0.00**. Open **`_data/products.json`** and set the real `price` for each product (numbers only, e.g. `"price": "49.99"`). You can also add/remove products, rename them, change categories, or set `"featured": true` to show one on the homepage. ### 2. Add your Snipcart public API key The cart is **inactive** until you add your key — so nothing can be bought by accident before you're ready. When you're set up: - Create a Snipcart account, then copy your **Public API Key**. - In **all four** HTML files (`index.html`, `shop.html`, `quality.html`, `contact.html`), find this line near the bottom: ```html
``` Replace `YOUR_SNIPCART_PUBLIC_API_KEY` with your real key. > **Payments:** Snipcart runs on top of a payment gateway. Peptides are a > restricted category for many mainstream processors, so connect Snipcart to > the **same processor you already use on aminos.science**. That's the part to > sort out first — the cart itself is ready. ### 3. Confirm the catalog matches your real store I built the catalog from the standard research-peptide lineup across the categories you named (Weight Loss, Anti-Aging, Injury Repair, Wellness). Please check it against your actual aminos.science products and add anything missing or remove anything you don't carry. The easiest way to hand me your exact list is a **WooCommerce export** (Products → Export in WordPress) — send it over and I'll drop your real names, prices, and descriptions straight in. --- ## Editing content Everything text-based lives in the **`_data/`** folder as simple JSON: | File | Controls | |------|----------| | `_data/site.json` | Company name, support email, footer text, legal disclaimer | | `_data/products.json` | The whole catalog + category filter buttons | | `_data/home.json` | Homepage hero, stats, and section copy | | `_data/quality.json` | The Quality page pillars | Edit the JSON, save, re-deploy. (There's also a `/admin` CMS panel, but it only works on a Git-connected Netlify site with Identity enabled — not with Netlify Drop. Editing the JSON directly is simplest.) ### Adding a product image Products show a clean placeholder vial graphic by default. To use a real photo, add an `"image"` field to that product, e.g.: ```json { "id": "bpc-157-5mg", "name": "BPC-157 5mg", ..., "image": "images/bpc-157.jpg" } ``` Put the file in the `images/` folder. --- ## Pages - **index.html** — Home - **shop.html** — Product catalog with category filters (also reachable as `shop.html?cat=weight-loss`) - **quality.html** — Purity & testing - **contact.html** — Contact form (submissions appear in your Netlify dashboard → Forms) ## Deploying Drag the whole folder onto **app.netlify.com/drop**. Done. ## Brand assets - `images/logo-white.png` — white logo for the dark nav/footer - `images/logo-ours-dark.png` — dark logo for light backgrounds - `images/site-icon.png` — favicon --- *All product copy is written at a research-compound level and the site carries a "research use only / not for human consumption" disclaimer throughout, consistent with how this product category is presented.*
